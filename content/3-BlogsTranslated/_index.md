This chapter presents a collection of blog posts translated from the **AWS Architecture Blog** and **AWS Blog**. These articles cover recent AWS technologies and best practices in cloud security, online gaming architectures, and Generative AI. By studying these blogs, readers can gain a better understanding of how AWS services are used to solve real-world challenges while building scalable, secure, and modern cloud applications.

### [Blog 1 - Dual-token Authentication for Nakama Game Servers with Amazon Cognito on AWS](3.1-Blog1/)
This blog explains how to implement **Dual-token Authentication** for Nakama game servers using **Amazon Cognito**. It describes how Cognito manages player identities while Nakama handles game sessions, and how services such as **Amazon CloudFront**, **AWS WAF**, **Application Load Balancer (ALB)**, **Network Load Balancer (NLB)**, and **Amazon ECS** work together to provide a secure, scalable, and seamless authentication experience for multiplayer games.


### [Blog 2 - Restricting AWS Management Console Access with Sign-in Resource-based Policies and RCPs](3.2-Blog2/)
This blog introduces **Sign-in Resource-based Policies** and **Resource Control Policies (RCPs)**, which enable organizations to restrict access to the **AWS Management Console** based on trusted network locations. It also explains how these features integrate with **AWS Organizations** and **AWS CloudTrail** to strengthen security, centralize policy management, and support auditing and compliance requirements.


### [Blog 3 - AWS GenAIIC Partner Agent Factory – Delivering AI Agents through AWS Marketplace](3.3-Blog3/)
This blog introduces the **AWS Generative AI Innovation Center (GenAIIC) Partner Agent Factory (PAF)**, a collaborative program that helps organizations deploy production-ready AI agents through **AWS Marketplace**. It highlights how **Amazon Bedrock**, **Amazon Bedrock AgentCore**, and **Strands Agents** are used to build enterprise AI solutions, enabling businesses to accelerate AI adoption while reducing development complexity and deployment time.
